Sobre mí

Soy Ingeniero en Electrónica Industrial y Automática, con formación en control de sistemas, instrumentación y desarrollo de software embebido. Mi TFG consistió en el diseño e implementación de una base de datos y entorno gráfico para la gestión de datos en simulación de transporte ferroviario, lo que me dio experiencia directa en modelado de datos, procesado de ficheros YAML y CSV, diseño de bases de datos en SQLite y construcción de interfaces funcionales con Tkinter. Actualmente curso formación en Inteligencia Artificial Aplicada a la Empresa, con foco en automatización de procesos, agentes y prompt engineering orientado a producción. Trabajo en los dos extremos del stack: desde C++ sobre microcontroladores para hardware embebido hasta Python para desarrollo de aplicaciones, automatización y entrenamiento de modelos.

TFG Python · SQLite
GitHub público
Diseño e implementación de una base de datos y entorno gráfico para la gestión de datos en la simulación de transporte ferroviario
Reto
El simulador de transporte ferroviario genera datos de salida en CSV a partir de configuraciones definidas en ficheros YAML independientes (oferta y modelado de demanda). Estos datos requerían persistencia estructurada, consultas eficientes y una interfaz de gestión accesible para perfiles no técnicos.
Pipeline de datos
La interfaz permite importar los archivos a la base de datos y reconstruir cualquier archivo original a partir de los registros almacenados.
Solución
Ingestión y parseo de ficheros YAML de configuración y CSV de resultados en Python, diseño del esquema relacional en SQLite y desarrollo de un entorno gráfico con Tkinter para la visualización, filtrado y gestión de los registros de simulación.
Python SQLite Tkinter YAML CSV SQL Modelado de datos

En desarrollo PyTorch · Audio ML
Síntesis Inversa con IA: Predicción de Presets de Vital desde Audio
Reto
Replicar un sonido en un sintetizador virtual requiere ajuste manual de decenas de parámetros interdependientes (osciladores, filtros, envolventes, efectos), un proceso costoso incluso para usuarios con experiencia en síntesis.
Solución
Modelo CNN entrenado sobre espectrogramas Mel que recibe una muestra de 2-5 segundos y predice los parámetros del sintetizador Vital como vector de regresión continua, generando un preset listo para cargar. Fase actual: construcción del generador de dataset con pares (audio renderizado → parámetros fuente) producidos programáticamente.
Estado actual
Generador de dataset funcional en desarrollo. Arquitectura CNN + cabeza de regresión multisalida definida. Entrenamiento pendiente de dataset suficiente.
PyTorch CNN Espectrograma Mel Python Síntesis de audio Regresión multisalida
En expansión ESP32 · RF 433 MHz
Hub de Control Climático con ESP32, DHT11 y Transmisión RF 433 MHz
Reto
Control y monitorización de temperatura y humedad en tiempo real sin infraestructura centralizada, con capacidad de actuar sobre un ventilador RF existente desde una interfaz web local, sin cableado adicional.
Solución
Nodo ESP32 con sensor DHT11 y emisor RF a 433 MHz para control del ventilador. Interfaz web embebida servida desde el microcontrolador, construida en HTML/CSS/JS con layout Flexbox. Expansión planificada: control PID o con lógica difusa para regulación de velocidad y temporizadores personalizados.
